## Trabajo 1 – Tipos de código en programación
**Objetivo**

Comprender los diferentes niveles de representación del código que se utilizan en programación y cómo se transforman hasta que el ordenador puede ejecutarlos.

**Contenidos trabajados**

- Código fuente

- Código objeto

- Código ejecutable

**Explicación**

### Código fuente
Es el código escrito por el programador utilizando un lenguaje de programación como Java, Python o C++. Este código es comprensible para los humanos pero necesita ser traducido para que el ordenador pueda ejecutarlo.

### Código objeto
Es el resultado de compilar el código fuente. El compilador transforma el código fuente en instrucciones que el ordenador puede procesar parcialmente.

### Código ejecutable
Es el archivo final que el sistema operativo puede ejecutar directamente. Contiene instrucciones en lenguaje máquina que el procesador puede interpretar.

**Conclusión**

La transformación del código fuente en código ejecutable es un proceso fundamental en el desarrollo de software, ya que permite que los programas escritos por los desarrolladores puedan ejecutarse en un ordenador.


## Trabajo 2 – Metodología Scrum
**Objetivo**

Conocer la metodología ágil Scrum y su uso en la organización y desarrollo de proyectos de software.

*¿Qué es Scrum?*

Scrum es una metodología ágil utilizada para gestionar proyectos de desarrollo de software de forma flexible y colaborativa. Se basa en la división del trabajo en pequeñas iteraciones llamadas sprints.

**Características principales**

- Trabajo en equipo

- Entregas frecuentes de funcionalidades

- Adaptación rápida a cambios

- Mejora continua del producto

- Roles en Scrum

- Product Owner
Representa al cliente y define los requisitos del producto.

- Scrum Master
Se encarga de que el equipo siga la metodología Scrum y elimina obstáculos que puedan aparecer.

- Equipo de desarrollo
Grupo de desarrolladores que construyen el producto.

**Elementos principales**

- Sprint: periodo de trabajo corto (normalmente de 1 a 4 semanas).

- Product Backlog: lista de tareas o funcionalidades del proyecto.

- Sprint Backlog: tareas que se realizarán durante el sprint.


**Conclusión**

Scrum permite organizar el trabajo de forma eficiente, mejorar la comunicación del equipo y adaptarse rápidamente a los cambios durante el desarrollo de software.
